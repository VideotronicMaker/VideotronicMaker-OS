# Contributing to VideotronicMaker-OS

Thank you for your interest in contributing to VTM OS! To ensure this project remains a high-quality, professional resource for the "Pro Maker" community, we have established the following guidelines.

All contributors must adhere to the **Code of Conduct**. We expect professional language, constructive feedback, and a spirit of appreciation for this open-source effort.

## üìñ Documentation-First & AI Support
To respect the time of the maintainers and the community, please follow these steps before submitting a contribution:
1. **Consult the Docs:** Read the README and all existing documentation thoroughly.
2. **Search:** Check existing Discussions and Issues to see if your topic has already been addressed.
3. **AI Troubleshooting:** Utilize AI assistants (such as Google Gemini, Claude, or ChatGPT) to attempt a fix or verify your logic before requesting manual intervention.

## üõ† How to Contribute
### Reporting Bugs or Requesting Features
Please use the provided **Issue Templates**. Issues that do not follow the template or ignore the pre-submission checklist may be closed to keep the project organized.

### Submitting Code (Pull Requests)
1. **Fork** the repository and create your branch from `main`.
2. **Hardware Testing:** You must test your changes on actual hardware (e.g., Uno Q) or the local development environment.
3. **Submit a Pull Request:** Use the VTM OS Pull Request template. Ensure your code is clean, modular, and contains no profanity or unprofessional comments.
